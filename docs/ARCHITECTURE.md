# Architecture

```text
raw input → model processing → J-space-like workspace readout → candidate commit → OpenLine witnesses → commit barrier → receipt chain
```

Internal interpretability can tell us what a model may be considering. Commit governance controls whether that internal state can alter external reality.

The dangerous step is when a thought becomes a tool call, file edit, memory write, external message, handoff, policy decision, or irreversible state change.

Design rule: separate workspace from commit.
